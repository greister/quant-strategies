## 1. 策略背景

### 1.1 动量效应

动量效应（Momentum Effect）是金融市场中广泛存在的现象，指过去表现较好的股票在未来一段时间内倾向于继续表现较好，而过去表现较差的股票则倾向于继续表现较差。

### 1.2 理论基础

- **行为金融学解释**: 投资者的反应不足和渐进信息扩散
- **基本面解释**: 盈利动量和商业周期
- **风险补偿解释**: 高动量股票承担更多风险

---

## 2. 因子定义

### 2.1 基础动量

```
Momentum_20D = (Price_today - Price_20d_ago) / Price_20d_ago × 100%
```

### 2.2 复合动量（当前实现）

```
Momentum_Score = 0.5 × Return_20D + 0.3 × Return_10D + 0.2 × Return_5D
```

### 2.3 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| 20日权重 | 50% | 中期趋势 |
| 10日权重 | 30% | 短期趋势 |
| 5日权重 | 20% | 近期动能 |

## 3. 实现方案

### 3.1 数据来源

| 数据 | 表名 | 字段 |
|------|------|------|
| 日K线 | raw_stocks_daily | open, high, low, close |
| 股票名称 | gtja_stock_names | name |
| 行业映射 | stock_industry_mapping | industry_code |

### 3.2 计算流程

```
Step 1: 获取当前价格
    ↓
Step 2: 获取20日、10日、5日前价格
    ↓
Step 3: 计算各周期收益率
    ↓
Step 4: 加权计算动量得分
    ↓
Step 5: 排名并存储结果
```

### 3.3 结果表结构

```sql
momentum_factor_daily
├── date: Date              # 日期
├── symbol: String          # 股票代码
├── name: String            # 股票名称
├── sector: String          # 行业
├── momentum_score: Float32 # 动量得分
├── return_1d/5d/10d/20d    # 各周期收益
├── rank: UInt32            # 排名
└── rank_pct: Float32       # 排名分位数
```

## 4. 使用场景

### 4.1 选股策略

```
每日收盘后:
1. 计算全市场动量因子
2. 选取 TOP 20（或前10%）
3. 次日开盘买入
4. 持有5-10天后调仓
```

### 4.2 行业轮动

```
按行业平均动量排序:
1. 选取动量最强的3-5个行业
2. 在每个行业内选取TOP标的
3. 实现行业+个股双重筛选
```

## 5. 风险控制

### 5.1 策略局限

| 局限 | 说明 | 应对 |
|------|------|------|
| 反转风险 | 高动量后可能反转 | 设置止损、缩短持有期 |
| 流动性风险 | 小盘股动量虚假 | 增加市值筛选 |
| 系统性风险 | 熊市中动量失效 | 结合市场趋势判断 |

### 5.2 建议参数

```
选股数量: 10-20只
持有周期: 5-10天
止损线: -5%
止盈线: +10%
```
