# 01. 独立强度因子 — 策略脚本完整目录

> **编号规则**: S=策略因子, A=分析工具, B=回测优化, C=报告可视化

## 一、核心因子策略 (S01~S05)

### S01 — 基础独立强度因子 (v2.0)
**脚本**: `calc_independence_score.sh`

**功能**:
- 基于 ClickHouse 5分钟K线，计算全A股相对所属板块的"逆势抗跌 + 顺势领先"得分
- **v2.0 升级**: 引入成交量加权 + 超额收益强度 + 板块同涨绝对强势
- 逆势得分 `contra_count` = Σ(区间成交额权重 × 逆势标志) × 48
- 顺势得分 `lead_count` = Σ(区间成交额权重 × 顺势领先标志) × 48
- 超额强度 `excess_strength` = Σ(超额收益% × 成交额权重 × 有效标志) / 10
- 总分 `score` = `contra_count` + `lead_count` + `excess_strength`
- 输出: `independence_score_daily` 表

**输入**: `raw_stocks_5min` + `stock_industry_mapping`

**优点**:
- 纯 ClickHouse SQL 计算，全市场约 3~5 秒完成
- 不依赖外部数据，每日可独立运行
- **v2.0**: 成交量加权过滤"无量抗跌"假信号，大额成交时段的逆势表现获得更高权重
- **v2.0**: 超额收益强度量化跑赢板块的程度，不再是简单的 0/1 计数
- **v2.0**: 板块同涨时的绝对强势也被计入（`lead_count`），弥补 v1.0 只关注抗跌的盲区

**不足**:
- 行业分类依赖 `stock_industry_mapping`，如果行业映射有误会污染得分
- 对早盘/尾盘极端情绪的时段区分度不足（需配合 S02 时间加权）
- `contra_count` / `lead_count` 已改为 Float64（成交量加权后不再是整数），下游若强转 Int 会截断小数
- v2.0 得分范围扩大至 0~48+，若下游使用固定阈值（如 ≥1.5）可能需要重新校准

---

### S02 — 时间加权独立强度因子
**脚本**: `calc_time_weighted_score.py`

**功能**:
- 在 S01 基础上引入时间衰减权重，不同时段赋予不同重要性
- 支持4种预设配置，通过 `score_weight_configs` 表管理权重
- 预设: `evening_focus`(尾盘关注型)、`morning_focus`(早盘关注型)、`trending_market`(趋势市)、`conservative`(保守均衡)
- 输出: `independence_score_time_weighted` 表

**输入**: `independence_score_daily` + `raw_stocks_5min` + `score_weight_configs`

**优点**:
- 可灵活配置交易时段偏好（如偏好尾盘强韧的股票）
- 权重配置持久化到数据库，支持历史回测不同配置
- 同一日可生成多个维度的选股名单

**不足**:
- 依赖 S01 数据，必须先运行 S01
- 权重配置依赖主观经验，缺乏自适应机制（不会根据市场环境自动调整权重）
- 4个预设无法覆盖所有市场环境（缺少"震荡市"、"恐慌市"等场景）
- 权重之和必须严格等于 1.0，配置错误会导致分数失真

---

### S03 — 融资余额加权独立强度因子
**脚本**: `calc_independence_score_margin_weighted.py`

**功能**:
- 将 S01 的独立强度分数与融资融券数据结合
- 加权公式: `weighted_score = raw_score * (1 + change_rate * weight_factor)`
- 融资余额增加的股票获得额外加分（默认 weight_factor=0.1）
- 输出: `independence_score_margin_weighted` 表

**输入**: `independence_score_daily` + PostgreSQL `margin_trading_detail_unified`

**优点**:
- 引入资金面验证，过滤"无量抗跌"的假信号
- 杠杆资金持续流入 + 独立强度高 = 双重确认
- 可调整加权系数适应不同风险偏好

**不足**:
- 依赖 PostgreSQL 融资融券数据，如果采集延迟则无法运行
- 深市融资偿还额通过余额差计算，存在利息计入误差
- 对高杠杆股票的加权可能放大风险（融资余额激增可能是被迫平仓前兆）
- 未区分融资买入 vs 融资偿还的持续性（单日突增 vs 连续流入）

---

### S04 — 高阶因子策略 (S09 / S10 / S12)
**脚本**: `calc_advanced_score.py`

**功能**:
- S09 — 黄金时段爆发检测: 10:30-11:30 出现 P99 拉升 + 30分钟不回落 + 放量 + 早盘抗跌
- S10 — 早盘抗跌+午后修复: 识别早盘弱势但午后放量修复的反转股
- S12 — 量能确认+两融验证: 成交额异常放大 + 融资/融券变化共振
- 输出: `independence_score_advanced` 表

**输入**: `raw_stocks_5min` + `independence_score_daily` + PostgreSQL 融资融券

**优点**:
- S09 对"盘中脉冲型"强势股识别精准，适合短线交易
- S10 捕捉反转机会，与 S01 形成互补（S01 抗跌 + S10 修复）
- S12 引入量能维度，过滤"缩量抗跌"的虚假强势

**不足**:
- S09 依赖 S01 的 `contra_count`，必须先运行 S01
- S10 的"午后修复"定义较主观（阈值固定，未考虑板块整体走势）
- S12 的量能异常检测使用固定阈值，对大盘股和小盘股的敏感度不一致
- 三个策略的参数（时间窗口、阈值）未经网格搜索优化，可能过拟合历史数据

---

### S05 — 周频一致性筛选 (S11)
**脚本**: `calc_weekly_consistency.py`

**功能**:
- 一周内 >= 3 天进入 S01 Top N 的个股，剔除"一日游"随机波动
- 评分维度: 入选天数、排名均值、得分稳定性(CV)、行业集中度
- 仅周五运行（需要近5个交易日数据）
- 输出: `weekly_consistency_screening` 表

**输入**: `independence_score_daily` (近5日)

**优点**:
- 过滤短期噪音，识别持续性强的独立股票
- 变异系数(CV)量化得分的稳定性，偏好"天天强"而非"某天极强"
- 周频视角适合中线持仓（5~10天持有期）

**不足**:
- 仅周五有意义，平日运行无数据
- 如果周内某日是节假日，有效天数减少，可能导致误判
- 对"周内逐渐走强"的股票不友好（前几天未入选，最后一天爆发）
- 未考虑周内板块轮动的结构性机会（如周初科技强、周末周期强）

## 二、分析选股工具 (A01~A03)

### A01 — 每日全市场选股扫描
**脚本**: `daily_stock_screening.py`

**功能**:
- 全市场批量扫描，输出增强因子综合评分排名
- 整合独立强度、融资融券、分时形态等多维度数据
- 生成 Top50 选股报告 + 融资异动股票列表
- 输出到 Obsidian Vault

**输入**: `independence_score_daily` + `raw_stocks_5min` + PostgreSQL 融资融券

**优点**:
- 一站式全市场扫描，每日收盘后5分钟内出结果
- 支持行业过滤、涨跌幅过滤、昨日对比
- 输出格式兼容 Obsidian，便于复盘

**不足**:
- 评分逻辑为阶梯加分制（非固定权重），不同维度间可比性较弱
- 未考虑市场情绪（如大盘涨跌对个股表现的基准影响）
- 对北交所股票的支持不完整（成交量和融资融券数据覆盖率低）
- 报告生成依赖文件系统IO，大量股票同时分析时较慢

---

### A02 — 个股多维度深度分析
**脚本**: `analyze_stock.py`

**功能**:
- 对单只股票做6维度深度分析: 分时形态、多日对比、行业对比、量能异动、两融分析、综合评分
- 输出详细的 Markdown 报告到 Obsidian Vault
- 支持多股票批量分析

**输入**: `raw_stocks_5min` + PostgreSQL 融资融券 + 行业数据

**优点**:
- 分析维度全面，适合对候选股做最终确认
- 可视化输出（VWAP偏离、量能分布、融资趋势图）
- 支持多日分时叠加对比，识别形态规律

**不足**:
- 单股分析耗时较长（约2~3秒/只），不适合全市场扫描
- Beta 计算使用5分钟收益率，样本量有限（48根K线），统计显著性较弱
- 行业对比仅取当日数据，未考虑行业的季节性/周期性变化
- 报告模板固定，无法根据股票特征自适应调整分析重点

---

### A03 — 三层筛选选股系统
**脚本**: `three_tier_screening.py`

**功能**:
- 第一层: 独立强度因子快速过滤（score ≥ threshold）
- 第二层: 对候选池计算增强因子群（VWAP / VaP / 杠杆 / Beta）
- 第三层: 五因子加权组合评分（IS30% + VWAP20% + 杠杆20% + Beta15% + VaP15%）
- 输出: Top N 选股名单 + 评分明细 + 买入建议

**输入**: `independence_score_daily` + `raw_stocks_5min` + PostgreSQL 融资融券

**优点**:
- 分层设计兼顾效率和深度，第一层3秒筛全市场，第二层精算Top候选
- 固定权重模型透明可解释，各因子贡献一目了然
- 支持 JSON 输出，便于下游策略自动调用

**不足**:
- Beta 计算基于当日5分钟K线 vs 沪深300，样本量小（~48对），Beta估计误差大
- VaP 计算依赖16/30的样本有效率，部分股票因数据缺失导致VaP为默认值
- 权重固定，未根据市场环境自适应调整（如熊市应提高Beta权重、牛市提高VWAP权重）
- 第一层 min-max 归一化基于30只候选池，如果候选池分布不均会导致归一化失真

---

## 三、回测优化工具 (B01~B03)

### B01 — 历史回测
**脚本**: `backtest_independence_score.py`

**功能**:
- 选取历史某日的独立强度高分股票作为买入信号
- 计算持有 N 天后的收益率、胜率、最大回撤、夏普比率
- 支持分板块统计

**输入**: `independence_score_daily` + `raw_stocks_5min` (未来价格)

**优点**:
- 快速验证因子在历史数据中的表现
- 支持参数扫描（阈值、持有期、Top N）

**不足**:
- 未考虑交易成本和滑点（实际收益会低于回测结果）
- 假设每日可满仓买入Top N，未考虑资金管理和仓位分配
- 对未来价格的查询使用 `raw_stocks_5min` 的日级别聚合，如果期间停牌会导致收益计算偏差
- 未考虑市场冲击（大额买入对价格的冲击成本）

---

### B02 — 参数优化
**脚本**: `optimize_backtest.py`

**功能**:
- 网格搜索独立强度因子的最优参数组合
- 测试维度: 选股阈值、持有天数、Top N 数量
- 输出各参数组合的胜率、夏普比率、最大回撤对比

**输入**: `independence_score_daily` (历史区间)

**优点**:
- 系统性寻找最优参数，避免主观设定
- 可视化参数热力图，直观展示参数敏感性

**不足**:
- 网格搜索的计算量随参数维度指数增长，3个以上参数时效率低
- 优化结果可能过拟合历史数据（在训练期表现好，未来表现差）
- 未做交叉验证（Walk-Forward Analysis），无法评估参数的稳定性
- 仅优化单一因子，未考虑多因子的协同效应

---

### B03 — 双因子组合演示
**脚本**: `combined_factor_demo.py`

**功能**:
- 演示如何将独立强度因子和动量因子结合
- 组合方式: S01 筛选后按动量排序取 Top N
- 支持不同市场环境的权重调整（牛市/熊市/震荡市）

**输入**: `independence_score_daily` + 动量数据

**优点**:
- 展示多因子组合的构建思路
- 代码简洁，适合作为教学示例

**不足**:
- 仅为演示脚本，未接入 02.momentum-factor 的真实动量数据
- 动量计算使用20日收益率，未考虑波动率调整（如夏普动量）
- 权重调整依赖人工输入市场环境，无自动判断机制
- 缺少回测验证，无法评估组合效果

## 四、报告可视化工具 (C01~C04)

### C01 — 策略执行报告生成
**脚本**: `gen_reports.py`

**功能**:
- 从 ClickHouse 读取各策略执行结果
- 生成 Obsidian Markdown 报告（含各策略 Top20、权重配置说明）
- 自动复制到 Vault 的 `策略执行结果/01-独立强度因子/` 目录

**输入**: `independence_score_daily` / `time_weighted` / `advanced` 等表

**优点**:
- 一键生成全策略报告，节省手动整理时间
- 报告格式统一，便于跨日对比

**不足**:
- 报告模板固定，无法根据当日市场特征动态调整内容
- 如果某策略未运行（如周五未运行 S11），报告中该策略为空，用户体验不佳
- 缺少图表（仅有表格），可读性不如可视化工具

---

### C02 — 市场统计
**脚本**: `market_stats.py`

**功能**:
- daily: 单日5分钟涨跌幅统计（分布、极端值、时段特征）
- weekly: 周统计（最近N个交易日的汇总）
- advanced: 高阶因子分析（波动率脉冲、动态Beta、反转效应）

**输入**: `raw_stocks_5min`

**优点**:
- 提供市场层面的宏观视角，辅助判断当日环境
- advanced 模式的波动率脉冲检测对识别市场转折点有帮助

**不足**:
- 统计报告偏重描述性，缺乏 actionable 的交易信号
- 周统计依赖最近N个交易日，如果期间有长假会导致样本偏差
- 高阶因子的解释性较弱，对非量化背景的用户不够友好

---

### C03 / C04 — 可视化
**脚本**: `visualize_independence_score.py` (matplotlib) / `visualize_independence_score_plotly.py` (Plotly)

**功能**:
- 独立强度得分的分布可视化（直方图、箱线图）
- 个股得分的时序趋势图
- 板块对比热力图

**优点**:
- 直观的图形展示，适合复盘和汇报
- Plotly 版本支持交互式查看

**不足**:
- 可视化脚本独立于选股流程，需要手动运行
- 未集成到每日自动化工作流中
- 对大量股票的渲染性能较差（>1000只时浏览器卡顿）

## 五、执行编排

### 编排脚本
**脚本**: `run_all_strategies.sh`

**执行顺序**:
```
并行启动: S01 + S02(evening_focus) + S02(morning_focus) + S02(trending_market) + S02(conservative) + S03
等待 S01 完成后启动: S04(S09/S10/S12)
等待全部完成后: C01(gen_reports) + C02(daily)
周五额外: S05 + C02(weekly) + C02(advanced)
```

**注意事项**:
- S04 依赖 S01，如果 S01 失败则 S04 也会失败
- S03 依赖 PostgreSQL 融资融券数据，如果当日未采集则自动跳过
- S05 仅周五运行，平日执行无效果
