## 二、策略研究

### 2.1 因子定义

#### 2.1.1 独立强度因子概念

独立强度因子是一个用于衡量个股在市场下跌环境中表现逆势抗跌能力的指标。该因子的核心逻辑在于：当所属板块经历显著下跌时，能够维持相对强势或者超额收益的个股，往往具备更强的内在价值支撑和资金关注度。

#### 2.1.2 计算逻辑

**触发条件：**
- 板块 5 分钟跌幅 < -0.5%

**计分规则：**

| 条件 | 计分 | 说明 |
|------|------|------|
| 个股涨幅 > 0% | +1 | 绝对收益表现优异 |
| 相对板块超额收益 > 1% | +1 | 显著跑赢板块 |

**公式表达：**
```
Independence Score = Σ(符合计分条件的 5 分钟区间数)

其中：
- 个股超额收益 = 个股涨幅 - 板块涨幅
- 板块跌幅阈值：-0.5%
- 超额收益阈值：1%
```

#### 2.1.3 因子特征

- **逆向选股**：专注于市场下跌时的相对强者
- **滚动累计**：通过全天累计消除单区间噪声
- **双重验证**：同时考虑绝对收益和相对收益

### 2.2 策略逻辑

#### 2.2.1 逆势抗跌的 Alpha 来源

**市场行为学解释：**

1. **情绪错配效应**
   - 板块下跌时，情绪性抛售往往导致优质个股被误杀
   - 能够抗跌的个股反映了市场对其内在价值的认可

2. **资金面信号**
   - 下跌环境中维持买盘需要更强的资金实力
   - 抗跌股票通常对应着机构资金的布局或利好预期

3. **信息优势**
   - 提前知晓利好的资金会在市场下跌时逆势布局
   - 这种信息优势将在后续市场修复中得到体现

#### 2.2.2 融资余额加权逻辑

**加权公式：**
```
Weighted Score = Raw Score × Margin_Weight

Margin_Weight = 1 + (融资余额变化率 × 0.1)
```

**加权逻辑说明：**

| 因素 | 说明 |
|------|------|
| 资金确认 | 融资余额增加表明有杠杆资金参与，增强信号可靠性 |
| 流动性筛选 | 融资标的通常具有较好的流动性，降低滑点成本 |
| 市场关注度 | 融资余额变化反映了市场对该股票的关注程度 |

#### 2.2.3 与 Margin Z-Score 因子的互补性

| 维度 | Independence Score | Margin Z-Score |
|------|-------------------|----------------|
| **信号类型** | 价格行为信号 | 资金面信号 |
| **触发时机** | 市场下跌期 | 无市场方向限制 |
| **核心逻辑** | 逆势抗跌能力 | 融资余额异动 |
| **组合效果** | 筛选逆势强者 | 资金面确认 |

**组合使用优势：**
- Independence Score 筛选出具有抗跌特质的个股
- Margin Z-Score 提供资金面的二次验证
- 两者结合可以提高信号的稳健性和可靠性

### 2.3 回测框架

#### 2.3.1 信号生成

**入场条件：**
```python
# 信号生成逻辑
def generate_signal(df, threshold=3):
    """
    生成交易信号

    Parameters:
    -----------
    df : DataFrame
        包含 Independence Score 和融资余额数据
    threshold : int
        入场阈值，默认为 3

    Returns:
    --------
    signal : Series
        1 表示入场，0 表示观望
    """
    # 计算融资加权后的得分
    df['weighted_score'] = df['independence_score'] * df['margin_weight']

    # 生成入场信号
    signal = (df['weighted_score'] >= threshold).astype(int)

    return signal
```

**阈值设置建议：**

| 市场环境 | 建议阈值 | 说明 |
|----------|----------|------|
| 低波动市场 | 3 | 提高阈值筛选更强信号 |
| 高波动市场 | 1-2 | 降低阈值捕捉更多机会 |
| 熊市环境 | 2 | 平衡信号数量与质量 |

#### 2.3.2 持有期收益计算

**收益计算流程：**

```python
def calculate_holding_return(df, signal_col='signal',
                              holding_period=5,
                              return_col='forward_return'):
    """
    计算持有期收益

    Parameters:
    -----------
    df : DataFrame
        价格和信号数据
    signal_col : str
        信号列名
    holding_period : int
        持有期（交易日），默认 5 日
    return_col : str
        收益率列名

    Returns:
    --------
    returns : Series
        每个信号对应的持有期收益
    """
    # 计算前向收益率
    df[return_col] = df['close'].pct_change(holding_period).shift(-holding_period)

    # 提取入场信号对应的收益
    entry_signals = df[df[signal_col] == 1].index
    returns = df.loc[entry_signals, return_col]

    return returns
```

**持有期设置：**
- 短线策略：3-5 个交易日
- 中线策略：10-20 个交易日
- 建议根据市场波动率动态调整

#### 2.3.3 评价指标

**核心指标体系：**

| 指标 | 公式 | 说明 |
|------|------|------|
| **胜率** | 赢利次数 / 总交易次数 | 收益为正的交易占比 |
| **平均收益** | Σ(收益率) / n | 每次交易的平均回报 |
| **夏普比率** | (平均收益 - 无风险利率) / 收益波动率 | 风险调整后收益 |
| **最大回撤** | max(1 - 净值 / max(净值)) | 历史最大亏损幅度 |
| **盈亏比** | 平均赢利 / 平均亏损 | 赢利与亏损的比率 |

### 2.4 使用场景

#### 2.4.1 选股过滤（每日 Top N）

**应用方式：**

```python
def daily_stock_selection(df, top_n=20, min_score=1):
    """
    每日选股过滤

    Parameters:
    -----------
    df : DataFrame
        全市场股票数据，包含 independence_score 和 margin_weight
    top_n : int
        选取最高分的 N 只股票
    min_score : int
        最小入选分数

    Returns:
    --------
    selected : DataFrame
        选中的股票列表
    """
    # 计算加权得分
    df['final_score'] = df['independence_score'] * df['margin_weight']

    # 过滤低分股票
    filtered = df[df['independence_score'] >= min_score]

    # 按得分排序，选取 Top N
    selected = filtered.nlargest(top_n, 'final_score')

    return selected[['stock_code', 'stock_name', 'independence_score',
                     'margin_weight', 'final_score']]
```

**使用建议：**
- 建议每日盘后运行，生成次日观察清单
- 结合流动性指标过滤（如日成交额 > 1亿）
- 可考虑行业分散配置，避免单一行业集中

```python
def sector_relative_ranking(df, sector_col='sector'):
    """
    板块内相对排名

    用于在特定板块内筛选相对强势个股

    result = []
```

#### 2.4.3 结合融资融券的确认信号

**多重确认框架：**

```python
def confirm_signal_with_margin(df,
                                 score_threshold=2,
                                 margin_change_threshold=0.05,
                                 margin_balance_threshold=100000000):
    """
    结合融资融券数据的信号确认

    Parameters:
    -----------
    df : DataFrame
        包含 independence_score 和融资融券数据
    score_threshold : float
        独立强度因子阈值
    margin_change_threshold : float
        融资余额变化率阈值（5%）
    margin_balance_threshold : float
        最小融资余额（1亿）

    Returns:
    --------
    confirmed_signal : Series
        经过融资融券确认的信号
    """
    # 基础信号
    base_signal = df['independence_score'] >= score_threshold

    # 融资余额条件
    margin_sufficient = df['margin_balance'] >= margin_balance_threshold
    margin_increasing = df['margin_change_pct'] >= margin_change_threshold

    # 融券余额条件（可选）
    short_not_increasing = df['short_change_pct'] <= 0

    # 综合信号
    confirmed_signal = (
        base_signal &
        margin_sufficient &
        margin_increasing &
        short_not_increasing
    ).astype(int)

    return confirmed_signal
```

**信号强度分类：**

| 信号等级 | 条件 | 建议操作 |
|---------|------|----------|
| 强信号 | Score >= 3 + 融资增加 + 融券减少 | 重仓布局 |
| 中信号 | Score >= 2 + 融资增加 | 适度参与 |
| 弱信号 | Score >= 1 + 融资稳定 | 观察为主 |
| 无信号 | 不满足上述条件 | 不操作 |

### 3.1 核心算法

#### 3.1.1 独立强度计算 SQL

```sql
-- 分时独立强度因子计算
-- 参数: {trade_date:Date}

WITH
-- 计算个股5分钟收益率
stock_returns AS (
    SELECT
        symbol,
        datetime,
        close,
        prev_close,
        (close - prev_close) / prev_close * 100 as stock_return
    FROM (
        SELECT
            symbol,
            datetime,
            close,
            lagInFrame(close) OVER (
                PARTITION BY symbol, toDate(datetime)
                ORDER BY datetime
                ROWS BETWEEN 1 PRECEDING AND CURRENT ROW
            ) as prev_close
        FROM raw_stocks_5min
        WHERE toDate(datetime) = {trade_date:Date}
    )
),

-- 获取股票板块归属
stock_with_sector AS (
    SELECT
        sr.symbol,
        sr.datetime,
        sr.stock_return,
        ss.sector_code
    FROM stock_returns sr
    INNER JOIN stock_sectors ss ON sr.symbol = ss.symbol
    WHERE sr.stock_return IS NOT NULL
),

-- 计算板块5分钟收益率
sector_returns AS (
    SELECT
        sector_code,
        datetime,
        avg(stock_return) as sector_return
    FROM stock_with_sector
    GROUP BY sector_code, datetime
),

-- 合并个股和板块收益率，计算逆势标志
combined_data AS (
    SELECT
        sws.symbol,
        sws.sector_code,
        sws.datetime,
        sws.stock_return,
        sr.sector_return,
        sr.sector_return < -0.5 AND (
            sws.stock_return > 0 OR
            (sws.stock_return - sr.sector_return) > 1
        ) as is_contra_move
    FROM stock_with_sector sws
    INNER JOIN sector_returns sr
        ON sws.sector_code = sr.sector_code
        AND sws.datetime = sr.datetime
)

-- 统计每个股票的逆势区间数量
SELECT
    symbol,
    sector_code,
    countIf(is_contra_move) as independence_score,
    count(*) as total_intervals
FROM combined_data
GROUP BY symbol, sector_code
```

#### 3.1.2 代码优化要点

| 优化项 | 实现方式 | 效果 |
|--------|----------|------|
| 窗口函数优化 | 使用 `lagInFrame` 替代多次计算 | 减少重复计算 |
| 条件聚合 | 使用 `countIf` 替代 CASE WHEN | 提高可读性和性能 |
| 子查询封装 | 将复杂逻辑拆分为 CTE | 便于维护和调试 |

### 5.2 参数配置

#### 5.2.1 阈值调整

| 参数 | 默认值 | 说明 | 调整建议 |
|------|--------|------|----------|
| `sector_return_threshold` | -0.5% | 板块下跌阈值 | 熊市可调至 -0.3% |
| `stock_return_threshold` | 0% | 个股上涨阈值 | 保守可设为 0.5% |
| `excess_return_threshold` | 1% | 超额收益阈值 | 激进可设为 0.5% |
| `weight_factor` | 0.1 | 融资加权系数 | 高风险偏好可调至 0.2 |

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--threshold` | 3.0 | 选股阈值 |
| `--hold-days` | 5 | 持有天数 |
| `--top-n` | None | 每日选股数量 |

### 5.3 常用查询

```sql
-- 某日独立强度排名
SELECT symbol, score, sector, contra_count

ORDER BY score DESC
LIMIT 20;

-- 某股票历史走势
SELECT date, score, contra_count
FROM independence_score_daily

SELECT symbol, score,
```

## 六、项目总结与展望

### 6.1 项目成果

#### 6.1.1 技术成果

- **ClickHouse 窗口函数实践**：熟练运用 `lagInFrame`, `countIf` 等高级函数
- **跨数据库集成方案**：实现 CH + PG 双库数据联动
- **量化因子框架**：建立可复用的因子计算和回测流程

#### 6.1.2 业务成果

- **新增量化因子**：独立强度因子可作为现有因子库的补充
- **策略验证工具**：回测框架支持快速验证因子有效性
- **可扩展架构**：易于添加更多加权维度（如北向资金、龙虎榜等）

### 6.2 核心优势

1. **逻辑清晰**：逆势抗跌的逻辑符合市场行为学原理
2. **数据驱动**：基于 5 分钟高频数据，信号更及时
3. **多维验证**：结合融资融券数据，提高信号质量
4. **灵活配置**：支持多种参数调整，适应不同市场环境

### 6.3 后续建议

#### 6.3.1 短期优化

- **历史数据批量回测**：验证因子在 2019-2024 年的表现
- **参数敏感性分析**：测试不同阈值组合的效果
- **与其他因子组合**：测试与动量、价值因子的协同效应

#### 6.3.2 中期扩展

- **实时监控面板**：使用 Grafana 展示每日因子分布
- **自动化交易接口**：对接交易系统实现自动下单
- **机器学习优化**：使用 XGBoost 优化权重系数

#### 6.3.3 长期规划

- **多因子模型**：构建包含 10+ 因子的综合评分模型
- **行业特化版本**：针对不同行业调整参数和逻辑
- **跨市场扩展**：适配港股、美股等其他市场

### 6.4 风险提示

- **过拟合风险**：因子在历史数据上表现好不代表未来有效
- **市场环境变化**：熊市和牛市的因子效果可能差异较大
- **数据质量依赖**：融资融券数据的准确性影响加权效果
- **流动性风险**：小盘股的高分信号可能存在执行难度
