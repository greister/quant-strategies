# 01. 独立强度因子 — 完整工作流

## 一、数据层

### 1.1 原始数据源

| 数据源 | 数据库 | 表/视图 | 说明 |
|--------|--------|---------|------|
| 5分钟K线 | ClickHouse | `raw_stocks_5min` | 全A股5分钟OHLCV |
| 行业分类 | ClickHouse | `stock_industry_mapping` | 通达信T级行业 |
| 融资融券-沪市 | PostgreSQL | `margin_trading_detail_sse` | 上交所个股明细 |
| 融资融券-深市 | PostgreSQL | `margin_trading_detail_szse` | 深交所个股明细 |
| 融资融券-统一 | PostgreSQL | `margin_trading_detail_unified` | 两市统一视图（含深市计算字段）|
| 融资融券排名 | PostgreSQL | `stock_margin_ranking` | 个股融资趋势/排名 |

### 1.2 数据更新时机

- **5分钟线**：交易时段实时写入 ClickHouse
- **融资融券**：T+1 披露，每天早上 09:15 由 `akshare-margin.timer` 自动采集
- **备份采集**：10:00 二次尝试（防止 09:15 失败）

## 二、因子计算层（8个策略）

### 2.1 执行方式

```bash
# 一键运行全部策略（并行）
./scripts/run_all_strategies.sh [日期]

# 或单独运行
./scripts/calc_independence_score.sh 2026-04-21           # S01
./scripts/calc_time_weighted_score.py 2026-04-21 --preset evening_focus  # S02
./scripts/calc_independence_score_margin_weighted.py 2026-04-21          # S06
./scripts/calc_advanced_score.py 2026-04-21 --strategy all               # S07
./scripts/calc_weekly_consistency.py 2026-04-21                          # S08(周五)
```

### 2.2 策略依赖关系

```
S01 基础独立强度
├──→ S02~S05 时间加权因子（依赖 S01 原始分数）
├──→ S06 融资加权（依赖 S01 + PG 融资融券）
├──→ S07 高阶因子（S09/S10/S12，依赖 S01 + PG 融资融券）
└──→ S08 周频一致性（S11，依赖近5日 S01）
```

### 2.3 输出表

| 策略 | 输出表 | 数据库 |
|------|--------|--------|
| S01 | `independence_score_daily` | ClickHouse |
| S02~S05 | `independence_score_time_weighted` | ClickHouse |
| S06 | `independence_score_margin_weighted` | ClickHouse |
| S07 | `independence_score_advanced` | ClickHouse |
| S08 | `weekly_consistency_screening` | ClickHouse |

## 七、常见问题

### Q1: 深市融资偿还额为什么为0？
- 深交所公开接口不披露个股融资偿还额
- 通过 `margin_trading_detail_unified` 视图自动计算：`prev_balance + buy - balance`

### Q2: 策略7（S09/S10/S12）为什么依赖策略1？
- S10 需要 S01 的 `contra_count` 作为输入
- 必须先完成 S01 计算，再运行 S07

### Q3: 为什么 08:00 采集融资融券会失败？
- 深交所 T+1 数据通常在 08:30~09:00 才发布
- 已调整为 09:15 执行
